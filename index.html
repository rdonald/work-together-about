WorkTogether.

An intuitive application that helps allieviate the painpoints of finding teamates and working with them in college classes. We allieviated these issues by leveraging the Flutter SDK, ASP.NET, MySQL, and Amazon Lightsail.

Find Teamates

Search and filter for compatible teamates by skills, interests, meeting times, and more!

Project Managment

Allocate tasks to specific teamates and plan out how your team will complete project milestones!

Professor Control

Professors can look into how team formation is going and control project goals!

Group Chatting

Communicate with potential teamates using our in app group messaging system!

Shared Functionality Tutorial

Users will begin by creating an account and logging in to the application. While not required, it is recommended to edit your profile by clicking the profile section of the navigation bar. By clicking the edit button, the user can change the fields to reflect who they are. These parameters are global for all classes and apply to both a professor and a student users.

Users will also be able to chat with any combination of students in their classes. To begin chatting, click the chat section of the navigation bar. Both professors and students use the same tool.

Clicking on create conversation allows the user to create a new conversation with as many users as they would like to invite. Clicking on already created conversations will allow you to send new messages. Chats are able to be re-named, and deleted at will by any user. When a chat is deleted the current user leaves the chat, but the remaining participants will still be able to use the conversation.

Student functionality Tutorial

Users that are students and have already created an account can navigate into a project and will be presented with a questionnaire of relevant fields that students find the most important. Once the questionnaire is submitted this will create an advertisement card for themselves for the rest of the class to see. The advertisement card will be a combination of both answers to the questionnaire and their profile page.

Once they are on the group search page a student can filter for specific fields at the top of the page, additionally they can enter into an alternative group search view where they can swipe through classmate cards to find suitable teammates. If a user finds a suitable teammate they can either swipe right on the card to invite them to their team or return to the group search page and click on their card and press the invite to team button.

To accept an invitation navigate to the notifications page and click on the notification and accept or reject. Once accepted if the user returns to the group search page they will see themself on a team and are automatically put into a team group chat on the chatting page.

If a student is in a project that has completed the group search phase they will now be in the project management phase where professors will assign active milestones to which the team has to work towards completing. The team can accomplish this by creating tasks and allocating them out amongst themselves.

Professor functionality Tutorial

In Work Together, any user can create a class and invite others to it. To create a class, the user must navigate to the home page and click on “Add/Delete Class”. After doing this, they will type a name and description for the new class and click Submit. After this is done, the class will appear on the user’s homepage.

Clicking on a classes project as the professor will bring up a dashboard containing project statistics, and from that page clicking on “Manage Project” will allow the professor to grab the classes invite code and move students between teams.
